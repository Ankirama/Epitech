\subsection{Detailed Description}
Include file for S\+D\+L quit event handling.

An \hyperlink{_s_d_l__events_8h_a3b589e89be6b35c02e0dd34a55f3fccaa31acc5fdafc86ebe2c1f5c3cae48d603}{S\+D\+L\+\_\+\+Q\+U\+I\+T} event is generated when the user tries to close the application window. If it is ignored or filtered out, the window will remain open. If it is not ignored or filtered, it is queued normally and the window is allowed to close. When the window is closed, screen updates will complete, but have no effect.

\hyperlink{_s_d_l_8h_a8fc8d35348d7c74bad8392d776c937b8}{S\+D\+L\+\_\+\+Init()} installs signal handlers for S\+I\+G\+I\+N\+T (keyboard interrupt) and S\+I\+G\+T\+E\+R\+M (system termination request), if handlers do not already exist, that generate \hyperlink{_s_d_l__events_8h_a3b589e89be6b35c02e0dd34a55f3fccaa31acc5fdafc86ebe2c1f5c3cae48d603}{S\+D\+L\+\_\+\+Q\+U\+I\+T} events as well. There is no way to determine the cause of an \hyperlink{_s_d_l__events_8h_a3b589e89be6b35c02e0dd34a55f3fccaa31acc5fdafc86ebe2c1f5c3cae48d603}{S\+D\+L\+\_\+\+Q\+U\+I\+T} event, but setting a signal handler in your application will override the default generation of quit events for that signal.
